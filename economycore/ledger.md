# 金流日誌

每筆銅錢 / 元寶異動都會寫入**稽核帳本**（`transaction_ledger`），可由管理員查詢、翻頁與退款。

## 查詢指令

| 指令 | 別名 | 權限（預設） |
|------|------|------|
| `/ledger <玩家> [起始日] [結束日] [頁碼]` | `/txlog` | `economy.ledger.view`（op） |

- 日期格式：`yyyy-MM-dd`。
- 預設查詢**最近 7 天**；每頁 **10 筆**。
- 區間上限 **90 天**，超過會自動向前截斷並於結果提示。
- 結果提供可點擊翻頁，滑鼠懸停可看單筆明細（含 operator、節點等）。

範例：

```
/ledger Steve                        # Steve 最近 7 天
/ledger Steve 2026-06-01 2026-06-15  # 指定區間
/ledger Steve 2026-06-01 2026-06-15 2  # 第 2 頁
```

## 帳本記錄什麼

每筆紀錄包含：交易 ID、幣種、類型、來源、發起 / 接收方、金額、雙方前後餘額快照、狀態、時間、處理節點、operator（若為遊戲內管理員操作）。

### 方向性

| 操作 | 方向 |
|------|------|
| 發放 | 系統 → 玩家 |
| 扣除 | 玩家 → 系統 |
| 轉帳 | 玩家 → 玩家 |

系統方以全零 UUID 表示且不記錄餘額快照；介面上系統方只顯示「系統」。

### 交易狀態

| 狀態 | 意義 |
|------|------|
| `SUCCESS` | 成功 |
| `FAILED` | 失敗（仍會寫入紀錄，便於稽核） |
| `REFUNDED` | 已被退款（原交易標記） |

## 設定

- `log-player-ip`（預設 `false`）：是否記錄發起玩家 IP。預設關閉，帳本與 Discord 告警皆不顯示 IP。
- `server-node-name`：多節點環境的節點識別名稱，寫入每筆紀錄。

退款相關請見 [管理指令](/economycore/admin)。
