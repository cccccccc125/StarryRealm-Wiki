# GUI 介面

拍賣行可**完全透過 GUI** 操作（上架、瀏覽、出價、購買、領取），指令為備援。共有六種介面：

| 介面 | 用途 |
|------|------|
| 瀏覽（BROWSE） | 主介面，瀏覽 / 篩選 / 排序所有掛單 |
| 詳情（DETAIL） | 單一掛單的出價 / 購買 / 下架 |
| 上架（SELL） | 放入物品、設定模式 / 時長 / 價格後上架 |
| 我的掛單（MY_LISTINGS） | 自己上架中的掛單 |
| 競拍追蹤（MY_BIDS） | 自己有出價的競拍 |
| 信箱（MAILBOX） | 領取得標 / 退回的物品 |

## 通用規則

- 所有拍賣行介面都以自訂 `InventoryHolder` 標記，點擊一律取消預設行為（不會被搬空），唯一例外是上架介面的「放入格」。
- 點擊後重活丟到背景（async）執行，完成再切回主執行緒刷新畫面，不卡主執行緒。

## 瀏覽（主介面）

`/ah` 開啟。上方為掛單格（分頁顯示），底部功能列提供：

- **類型篩選**：依物品分類（`ItemCategory`）循環。
- **品質篩選**：依 MMOItems 品質（tier）循環。
- **排序**：品質高→低、價格↑、價格↓、名稱 A-Z、剩餘時間。
- **上一頁 / 下一頁**、中央資訊（餘額 / 頁碼 / 點擊重新整理）。
- **＋ 上架**：開啟上架介面。
- **我的掛單 / 競拍追蹤 / 信箱**：開對應介面（信箱有未領取數量提示）。

## 詳情

點掛單進入。中央顯示物品，下排按鈕：

- **出價**（競拍）：開**鐵砧**輸入出價金額（預填最低可出價）。
- **立即購買**（一口價）：付款購得。
- **下架**：僅賣家本人可見（競拍須無人出價）。

## 上架介面（27 格）

由主介面「＋ 上架」開啟。slot 配置：

| slot | 功能 |
|------|------|
| 10 | **放入格**（把要賣的物品放進來，可放 / 取） |
| 12 | **模式鈕**（競拍 / 一口價，點擊就地切換） |
| 14 | **時長鈕**（循環 12 / 24 / 48 / 72 / 168 小時） |
| 16 | **價格鈕**（顯示目前價或「未設定」，點擊開鐵砧輸入） |
| 22 | **確認鈕**（驗證後上架） |
| 26 | **取消鈕**（關閉並退回物品） |

### 鐵砧金額輸入

設定價格 / 出價時會開啟鐵砧（AnvilGUI）輸入數字：輸入**正數**後點輸出格確認；輸入無效會原地提示重輸。鐵砧往返期間上架介面的物品**不會遺失**。

### 物品安全

- 關閉上架介面（非確認成功）→ 放入格物品退回背包（滿則掉落腳邊）。
- 上架介面開啟中離線 → 放入格 / 暫存物品投進**信箱**（`LISTING_RETURNED`），不會遺失。
- 確認成功 → 物品已交給上架流程，介面關閉時放入格已空。

## 出價（全 GUI）流程

詳情「出價」→ 鐵砧輸入金額 → 確認 → 背景送出出價 → 回饋並刷新詳情。等同於指令 `/ah bid`，規則見[競標](/economycore/auction/bidding)。

> AnvilGUI 函式庫已 shade + relocate（`com.economycore.libs.anvilgui`），不會與其他插件衝突。
