# 上架

把物品放上拍賣行，可選**競拍**或**一口價**。需要權限 `auction.sell`（預設 true）。

## 方法一：GUI（推薦）

1. `/ah` 開主介面 → 點功能列的 **「＋ 上架」** 開上架介面（27 格）。
2. 把要賣的物品**放進放入格**（左上的可放/取格）。
3. 點 **模式鈕** 切換「競拍 / 一口價」、點 **時長鈕** 循環 12 / 24 / 48 / 72 / 168 小時（就地切換，不會關閉介面）。
4. 點 **價格鈕** → 開**鐵砧**，輸入價格（需為正數）後確認。
5. 點 **確認鈕** 完成上架；點 **取消鈕** 或關閉介面會把物品退回。

> 鐵砧設價期間介面會短暫切換，物品**不會遺失**；若中途離線，放入格的物品會自動投進你的信箱（`LISTING_RETURNED`）。

## 方法二：指令

```
/ah sell <auction|buyout> <價格> [時長(小時)]
```

上架的是**主手手持**的物品。例如：

```
/ah sell buyout 5000        # 一口價 5000 銅錢
/ah sell auction 1000 72    # 競拍，起標 1000，72 小時
```

執行後物品會立刻從手上移除；若上架失敗則退回。

## 上架規則與驗證

| 條件 | 說明 |
|------|------|
| 物品 | 必須非空（手持或放入格有物品） |
| 價格 | 必須 > 0（競拍為起標價，一口價為成交價） |
| 時長 | 介於允許範圍；省略用預設（預設 48 小時，上限 168 小時） |
| 掛單數 | 每人同時上架數有上限（預設 10），達上限無法再上架 |

對應的失敗原因：

| 結果 | 意義 |
|------|------|
| `INVALID_ITEM` | 物品無效 |
| `INVALID_PRICE` | 價格必須大於 0 |
| `INVALID_DURATION` | 時長超出允許範圍 |
| `TOO_MANY_ACTIVE` | 掛單數已達上限 |
| `DB_ERROR` | 系統錯誤 |

## 物品如何被保存

上架時物品會以 Base64 完整序列化（保留 lore、附魔、PDC 等），存入掛單。有安裝 **MMOItems** 時會解析其 tier（品質）與 type（分類）供瀏覽篩選 / 排序；無 MMOItems 則品質為空、依材質概略分類。
