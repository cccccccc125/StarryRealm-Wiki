# 綁定與驗證流程

## 首次綁定（/2fa enable）

只有 **OP** 能啟用 2FA。

1. 執行 `/2fa enable`。
2. 系統產生 TOTP 密鑰，並把 QR Code 渲染到一張**地圖**交給你。
3. 用驗證器 App（Google / Microsoft Authenticator…）**掃描地圖上的 QR Code**。
4. 輸入驗證碼完成綁定：
   - **聊天輸入模式**（預設）：直接在聊天框輸入 App 顯示的 6 位數驗證碼。
   - **安全輸入模式**（`secure-input: true`）：掃描後**再次執行 `/2fa enable`**，會開啟鐵砧視窗輸入驗證碼（驗證碼不會出現在公開聊天 / proxy 紀錄）。
5. 驗證成功即啟用。

> 為避免密鑰外洩，QR Code 地圖若在逾時時間內（預設 120 秒）未完成綁定會被自動移除，需重新執行 `/2fa enable`。

## 登入驗證（已綁定的 OP 加入時）

1. OP 加入伺服器時會被**凍結**（無法移動 / 互動），加入訊息隱藏。
2. 依設定提示在**聊天框**或**鐵砧視窗**（`secure-input: true`）輸入當下的 6 位數驗證碼。
3. 驗證成功 → 解凍，正常遊玩；觸發 [`TwoFAVerificationEvent`](/twofactorauth/events)（SUCCESS）。
4. 驗證失敗 → 提示剩餘嘗試次數；連續失敗達上限（預設 3 次）→ 帳號**鎖定**一段時間（預設 5 分鐘）。鎖定與失敗計數存於資料庫、**重連也不會重置**。
5. 驗證逾時（預設 300 秒未完成）→ 需重新加入伺服器。

## 停用（/2fa disable）

執行 `/2fa disable` 停用自己的 2FA（流程同樣需確認，避免誤觸）。

## 管理員重置（/2fa reset）

具 `twofactorauth.2fa.reset` 權限者可 `/2fa reset <玩家>` 重置他人的 2FA（清除綁定與凍結 / 鎖定）。為防止繞過，執行者本身若尚未通過驗證（仍被凍結）則不可使用。

相關指令見 [指令](/twofactorauth/commands)。
