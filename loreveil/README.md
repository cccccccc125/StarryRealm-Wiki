# LoreVeil

讓玩家**自行切換物品詳細描述（lore）顯示/隱藏**的視覺插件。RPG 伺服器的裝備 lore 往往很長，擋住畫面；玩家輸入 `/lore` 即可把所有物品的 lore 收合成一行「物品敘述已隱藏」，再輸入一次即恢復完整顯示。

## 特色

- **玩家層級偏好**：每位玩家獨立切換，互不影響；偏好存於玩家 PDC，離線重登、跨重啟都保留。
- **純視覺、零風險**：採**封包端改寫**——只改「送往客戶端」的顯示副本，真實物品永不改動。不需備份 lore、不需與 MMOItems 協調，也沒有丟棄/交易/序列化的邊界問題。
- **容器 GUI 不受影響**：開啟箱子、自訂選單時，容器格子照常顯示完整 lore，只有玩家自己的背包格子套用隱藏。
- **創造模式安全**：創造客戶端回寫的顯示副本會在伺服器端自動還原成真實物品，污染在結構上不可能發生。
- **格式完整保留**：lore 以 Adventure 元件 JSON 備份，RGB 顏色、自訂字型、hover 事件經隱藏/還原後完整保留。

## 運作概觀

```
玩家輸入 /lore（切換為隱藏）
   │ 偏好寫入 PDC + 記憶體快取，立即重送背包封包
   ▼
伺服器 → 客戶端的物品封包（SET_SLOT / WINDOW_ITEMS）
   │ 攔截：把有 lore 的物品換成「顯示副本」
   │        （lore 只剩一行「物品敘述已隱藏」，內嵌原始 lore 備份）
   ├─ 容器格子（箱子 / 自訂 GUI）→ 不碰，照常顯示
   └─ 玩家背包格子 → 套用隱藏
   ▼
客戶端 → 伺服器（創造模式 SET_CREATIVE_SLOT）
   └─ 若是顯示副本 → 用內嵌備份還原成真實物品後放行
```
